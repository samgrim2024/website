Public Group

Problem

Salesforce’s native permission model blocks project managers from adding or removing users from project public groups, creating unnecessary admin dependency.

Solution

Run as a System-context screen flow.

Solution

  • Used GroupMember as the junction object to add and remove members by GroupMember.Id instead of User.
  • Transformed record collections into text collections to support efficient IN (Public Group)and NOT IN queries.
  • Validated that users were selected before processing.
  • Configured related lists + list views to make day-to-day tracking easy.
  • Only added or removed GroupMembers after an explicit confirmation step to prevent accidental changes.
